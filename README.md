
# Сравнение конвейеров предобработки данных: Python/Pandas vs Haskell

Данный репозиторий содержит исходный код к исследованию, посвященному сравнению надежности конвейеров предварительной обработки данных (Data Preparation). В проекте сопоставляется традиционный скрипт на Python (Pandas) и строго типизированный конвейер на языке Haskell, реализующий парадигму «Parse, don't validate».

## Структура проекта

```text
├── Haskell/
│   └── DataPreparation.hs     # Типобезопасный конвейер на Haskell
├── Python/
│   ├── csvgenerator.py        # Генератор синтетического "грязного" датасета
│   └── data_preparation_python.py # Скрипт предобработки на Python/Pandas
└── .gitignore                 # Файл исключений Git
```

---

## Требования к окружению

Для запуска проекта на локальном компьютере должны быть установлены:
1. **Python** (версии 3.10 или выше).
2. **Компилятор GHC** (Glasgow Haskell Compiler, рекомендуется версия 9.2 или выше). Проще всего установить его через [GHCup](https://www.haskell.org/ghcup/).

---

## Инструкция по запуску

### Шаг 1. Подготовка Python-окружения и установка библиотек

Перейдите в папку `Python`, настройте виртуальное окружение и установите необходимые зависимости (`pandas` и `numpy`):

```bash
# Переход в рабочую директорию Python
cd Python

# Создание и активация виртуального окружения (опционально, но рекомендуется)
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для macOS/Linux:
source venv/bin/activate

# Установка зависимостей
pip install pandas numpy
```

### Шаг 2. Генерация «грязного» датасета

Запустите скрипт генерации данных. Он создаст файл `dirty_dataset.csv` в папке `Python`, внедрив туда 5% пропусков, 4% некорректных типов данных, опечатки в категориях и нарушения бизнес-логики:

```bash
python csvgenerator.py
```
*После успешного выполнения в папке появится файл `dirty_dataset.csv`.*

### Шаг 3. Запуск предобработки на Python (Pandas)

Запустите скрипт очистки данных на Python:

```bash
python data_preparation_python.py
```

**На что обратить внимание в выводе:**
* Скрипт покажет, сколько строк «потеряно» (отсекаются только NaN-значения).
* Обратите внимание, что 20 строк со сломанными статусами (`activ`, `UNKNOWN`) удаляются Pandas молча, проходя мимо счётчика отслеживания потерь.
* Логическая ошибка (отрицательный доход `-500`) пропускается скриптом в финальный набор данных без предупреждений.

---

### Шаг 4. Запуск предобработки на Haskell

Перейдите в папку `Haskell`. Перед компиляцией убедитесь, что путь к CSV-файлу в коде `DataPreparation.hs` соответствует вашей структуре. В коде по умолчанию прописано:
`csvData <- readFile "../../Python/CSV/dirty_dataset.csv"` 
*(При необходимости измените этот путь на `"../Python/dirty_dataset.csv"` согласно вашей структуре папок).*

Скомпилируйте и запустите Haskell-версию:

```bash
# Переход в рабочую директорию Haskell
cd ../Haskell

# Компиляция кода с оптимизацией
ghc -O2 DataPreparation.hs

# Запуск скомпилированного бинарного файла
# На Windows:
DataPreparation.exe
# На macOS/Linux:
./DataPreparation
```

**На что обратить внимание в выводе:**
* Haskell-конвейер не производит «тихих» удалений: на этапе структурного разбора (pattern matching) отлавливаются все аномалии (60 строк).
* На экран выводится подробный лог первых 5 ошибок с точным указанием причин (ошибки типизации в конкретных полях, невалидные статусы).
